User: "Tell me more about authentication"

→ Previous context helps Claude understand it's same course


================================================================================
                          🎯 KEY DESIGN PATTERNS
================================================================================

1. TOOL-BASED SEARCH
   • AI decides when to search (not hardcoded)
   • Can search 0, 1, or 2 times per query
   • Flexible: course filter, lesson filter, or both

2. TWO-COLLECTION STRATEGY
   • course_catalog: Fast course name resolution
   • course_content: Semantic content search
   • Enables partial name matching ("Anthropic" → full title)

3. SOURCE ATTRIBUTION
   • Every answer tracks which chunks were used
   • Lesson links provided for verification
   • User can click through to original content

4. SESSION MANAGEMENT
   • Stateful conversations (remembers context)
   • Limited to 2 exchanges (prevent token bloat)
   • Server-side storage (not client cookies)

5. MARKDOWN RENDERING
   • Backend returns plain markdown
   • Frontend converts to HTML
   • Supports code blocks, lists, formatting
